- [ ] Screen resolution set to 1920x1080 or 1280x720

## SCRIPT

### [0:00-0:15] INTRODUCTION (15 seconds)

**[Show app homepage]**

**Say:**
"Hi, I'm [Your Name], and this is the Mini RAG Assistant - a Retrieval-Augmented Generation system that answers questions using your local documents. Let me show you how it works."

**Action:**
- Show clean app interface
- Cursor on main question input

---

### [0:15-0:35] DOCUMENT LOADING (20 seconds)

**[Show sidebar with document info]**

**Say:**
"The system automatically loads documents from the docs folder. Here we have four company policy documents already indexed - that's about 50 chunks ready for semantic search."

**Action:**
- Point to sidebar showing "Documents Loaded: 4"
- Briefly show the docs folder in file explorer (optional)
- Return to app

---

### [0:35-1:20] SINGLE QUERY DEMO (45 seconds)

**[Type first question]**

**Say:**
"Let's ask: 'What is the vacation leave policy?'"

**Action:**
- Type the question slowly so viewers can read
- Click "Ask" button
- Wait for response

**[Show response]**

**Say:**
"The system retrieves relevant context, generates an answer, and provides an 87% confidence score. Below, we can see the exact source documents used - here's a chunk from the company policy with a 92% relevance score."

**Action:**
- Point to confidence score
- Scroll to sources section
- Expand one source to show content
- Point to relevance score
- Briefly show the document name

---

### [1:20-2:05] MULTI-TURN CONVERSATION (45 seconds)

**[Type follow-up question]**

**Say:**
"Now watch this - I'll ask a follow-up: 'How do I request it?' Notice I'm using the pronoun 'it' without specifying vacation leave."

**Action:**
- Type "How do I request it?"
- Click "Ask"
- Wait for response

**[Show response]**

**Say:**
"The system understands from context that 'it' refers to vacation leave, and correctly explains the request process. Let's try one more: 'What if I need to cancel?'"

**Action:**
- Type "What if I need to cancel?"
- Click "Ask"
- Show response

**Say:**
"Perfect - it maintains the conversation context and answers about canceling vacation requests."

---

### [2:05-2:35] TEXT HIGHLIGHTING (30 seconds)

**[Scroll to sources]**

**Say:**
"One unique feature is intelligent text highlighting. The system highlights exact keyword matches in yellow and semantically similar phrases in green, making it easy to verify the information."

**Action:**
- Expand a source with highlighted text
- Point to yellow highlights (exact matches)
- Point to green highlights (semantic matches)
- Scroll through to show multiple highlights

---

### [2:35-2:50] PROVIDER SWITCHING (15 seconds)

**[Show sidebar]**

**Say:**
"The system supports both OpenAI for high quality and Ollama for local, private deployment. You can switch between them anytime."

**Action:**
- Point to provider selection radio buttons
- Optionally switch to Ollama (if running)
- Show that it maintains separate vector stores

---

### [2:50-3:00] WRAP-UP (10 seconds)

**[Show full interface]**

**Say:**
"That's the Mini RAG Assistant - accurate, transparent, and flexible. Thanks for watching!"

**Action:**
- Show full app one more time
- Fade out or end recording

## RECORDING TIPS

### Technical Setup:
- **Tool:** Loom (easiest), OBS Studio (free), or built-in screen recorder
- **Resolution:** 1280x720 minimum, 1920x1080 preferred
- **Frame Rate:** 30 fps
- **Audio:** Use headset microphone for clear audio
- **Browser:** Chrome or Edge (clean interface)

### Recording Best Practices:
1. **Close unnecessary apps** - Clean desktop
2. **Disable notifications** - Focus mode on
3. **Full screen browser** - F11 in most browsers
4. **Zoom if needed** - Ctrl/Cmd + to make text larger
5. **Slow down** - Speak clearly, don't rush
6. **Pause between sections** - Easier to edit later
7. **Have water nearby** - Stay hydrated
8. **Do a test recording** - Check audio/video quality

### Speaking Tips:
- **Pace:** Speak slightly slower than normal
- **Energy:** Keep enthusiasm up (smile while talking)
- **Clarity:** Enunciate clearly
- **Pauses:** Brief pause after each feature
- **Confidence:** You built this - be proud!

### Common Mistakes to Avoid:
- ‚ùå Talking too fast
- ‚ùå Saying "um" or "uh" too much
- ‚ùå Apologizing for anything
- ‚ùå Going over 3 minutes
- ‚ùå Forgetting to show key features
- ‚ùå Poor audio quality
- ‚ùå Cursor moving too fast
